# Task 3 — Agentic Financial Research Workflow

## Overview

This project implements an observable multi-agent financial research workflow designed for production-style AI systems engineering.

The workflow combines:
- financial market data analysis
- news retrieval
- LLM-based sentiment analysis
- role-based agent orchestration
- critique/review loops
- persistent workflow memory
- execution trace logging

The architecture prioritizes:
- explainability
- structured outputs
- graceful degradation
- observability
- notebook-friendly orchestration

rather than highly autonomous black-box agent systems.

---

## Workflow Architecture

```text
User Ticker Input
        ↓
Financial Tools Layer
- get_price_data()
- calculate_volatility()
- get_news()
- llm_sentiment()
- web_search()
        ↓
Quantitative Analyst Agent
        ↓
Sentiment Research Agent
        ↓
Risk Review / Critique Agent
        ↓
Portfolio Strategist Agent
        ↓
Structured Final Report
        ↓
Persistent Memory Cache
        ↓
Follow-Up Question Handling
```

## Engineering Notes

The workflow intentionally uses explicit orchestration instead of autonomous planning loops.

This improves:
- debuggability
- reliability
- observability
- structured validation
- interview explainability

Each agent has a focused responsibility and communicates through validated structured outputs.
